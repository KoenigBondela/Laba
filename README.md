## Консольное приложение для настроек оборудования

Простое консольное приложение на **Java + Maven + AspectJ**.  
В зависимости от роли пользователя (`user` или `root`) показывается своё меню.

---

### Что делает программа

- Сначала программа просит ввести логин и пароль.
- Есть два стандартных пользователя:
  - логин `user`, пароль `user123` – обычный пользователь;
  - логин `root`, пароль `root123` – администратор.
- После успешного входа в систему:
  - для **user** меню:
    1. посмотреть настройки оборудования;
    2. посмотреть статистику;
    3. выход;
  - для **root** меню:
    1. изменить настройки оборудования;
    2. посмотреть статистику;
    3. добавить нового пользователя;
    4. выход.

Дополнительно, с помощью AspectJ в консоль выводятся простые логи (например, попытка авторизации, открытие меню, изменение настроек).

---

### Как устроен проект (по папкам)

- **`pom.xml`**
  - обычный Maven-файл с настройками;
  - подключены библиотеки AspectJ и плагин для их работы;
  - настроен запуск `main`-класса через `mvn exec:java`.

- **`org.example.app.Main`**
  - главный класс;
  - читает логин/пароль;
  - определяет роль (user или root);
  - в зависимости от роли выбирает нужное меню и запускает его.

- **`org.example.auth`**
  - `Role` – перечисление с ролями `USER` и `ROOT`;
  - `AuthService` – проверяет логин и пароль, умеет добавлять новых пользователей.

- **`org.example.menu`**
  - `MenuStrategy` – общий интерфейс для всех меню (один метод `showMenu`);
  - `MenuContext` – класс, который хранит текущую стратегию меню и запускает её;
  - `UserMenuStrategy` – меню для обычного пользователя (`user`);
  - `RootMenuStrategy` – меню для администратора (`root`).

- **`org.example.services`**
  - `EquipmentConfigService` – «настройки оборудования» (просмотр и изменение строки с конфигурацией);
  - `EquipmentStatsService` – «статистика оборудования» (просто печатает примерные значения).

- **`org.example.aspects.LoggingAspect`**
  - аспект AspectJ, который выводит в консоль сообщения о:
    - попытке и результате авторизации;
    - открытии меню;
    - изменении настроек;
    - добавлении нового пользователя.

---

### Как запустить

Нужны **JDK 17** и **Maven**.

Из корня проекта:

```bash
mvn clean package
mvn exec:java
```

Дальше следуйте подсказкам в консоли (введите логин/пароль, выбирайте пункты меню).

---

### Какой паттерн использован и зачем

Здесь используется паттерн **Strategy (Стратегия)**.

- Есть общий интерфейс `MenuStrategy` – описывает, как показывать меню и обрабатывать команды.
- Для разных ролей есть разные реализации:
  - `UserMenuStrategy` – меню для `user`;
  - `RootMenuStrategy` – меню для `root`.
- Класс `MenuContext` просто хранит выбранную стратегию и запускает её, не зная деталей.

**Почему это удобно:**

- Меню легко менять и расширять: можно добавить новую роль и новое меню, почти не трогая остальной код.
- Логика авторизации (`AuthService`), логика выбора меню (`Main` + `MenuContext`) и сама работа меню разделены и не мешают друг другу.

AspectJ используется только для логирования – чтобы не писать в каждом методе `System.out.println`, а собрать логи в одном месте (`LoggingAspect`).
